## 현재 구현 범위

### 완료 또는 사용 가능

| Phase | 상태 | 구현 내용 |
|---|---|---|
| 0. 기존 기능 안정화 | 부분 완료 | 기존 Google 로그인·자소서 저장을 유지했고 `npm run typecheck`, `npm run build`를 통과합니다. |
| 1. 경험 아카이브 골격 | 완료 | 경험 아카이브, 검색, 카테고리·상태 필터, 카드 목록, 빠른 메모와 단계별 기록 진입을 제공합니다. |
| 2. 경험카드 DB/이전 | 완료 | 카드·성과·역량·스토리 테이블과 사용자 소유권 검증을 추가했습니다. 기존 `experiences`는 보존하고 신규 카드의 원문 메모로 복사합니다. |
| 3. 경험 작성 | 부분 완료 | 기본 정보, 원문 메모, 상황·과제·행동·결과·배운 점, 성과, 역량, 태그를 저장합니다. 브라우저 임시 저장과 저장되지 않은 변경 이탈 경고를 제공합니다. |
| 4. 여러 스토리 각도 | 부분 완료 | 카드별 스토리 각도 생성·저장 및 대표 스토리 표시 기반을 제공합니다. |
| 7. 문항-경험 수동 연결 | 완료 | 문항별 핵심 경험 1개·보조 경험 2개를 연결하고, 스토리 각도와 핵심 사실·확인 수치를 확인할 수 있습니다. |
| 문항 예시 | 완료 | 지원 동기·직무 역량·문제 해결·협업·실패와 개선 예시 문항을 새 자소서 화면에서 바로 추가할 수 있습니다. |
| 8. AI 분석 (경험 보강 질문) | 완료 | 경험카드의 `AI 분석` 버튼을 누르면 모달에서 저장된 원문을 바탕으로 AI가 판단한 추가 질문을 하나씩 던집니다. 답변·건너뛰기·중단이 가능하며, 답변한 내용을 종합해 원문 메모를 보강하고 자동 저장합니다. 기존의 필드별 구조화 제안·적용 방식(AI 정리하기)은 제거했습니다. |
| 11. AI 자소서 초안 | 완료 | 문항에 직접 연결한 경험카드·스토리·확인된 성과만 근거로 AI 초안을 만들고, 새 답변 버전으로 저장합니다. 사용자가 적용하기 전에는 기존 답변을 바꾸지 않습니다. `AI 초안 만들기`는 먼저 모달에서 답변 흐름(개요)을 입력받고, AI가 충분성을 판단해 부족하면 추가 질문을 던진 뒤 초안을 생성합니다. 추가 질문 답변은 핵심 경험카드의 원문 메모로 승격되어 재사용됩니다. 문항별 기업 정보(인재상·정책·신년사 등) 입력도 지원합니다. 초안은 실제 우수 자소서 5건(삼일 Deals/Audit, 삼정 Audit, 안진 Audit, EY한영 Audit)을 벤치마크해 문체를 다듬었습니다: `[소제목]`(짧은 정체성 문구 또는 완결 주장 문장)으로 시작하고, 단일 주제 문항은 훅(직무·산업 관찰)·본론(구체 경험·수치·기업 정보 연계)·결론(단기·장기 다짐) 구조로, 하위 질문이 여러 개 묶인 문항은 일반론 도입 + `[소제목]` 경험 블록 2개(각각 기업·부문명 연결 문장으로 마무리)로 자동 분기합니다. 소통·협업 등 소프트스킬 경험에도 구체적 수치를 반영하고, 기업 정보는 막연한 형용사가 아니라 구체적 고유명사(툴·프로그램·가치명)로 연결하도록 지시합니다. 글자 수·byte 제한의 90%까지 채우도록 시도하며 부족하면 자동으로 한 번 더 보강합니다. 해시태그 형식을 요구하는 문항(예: "해시태그로 작성")은 자동 감지해 `#키워드` 형식으로만 짧게 작성합니다. |
| UI 단순화 | 완료 | 일정·좌측 최근 기록을 화면에서 제거하고, 상단 `경험 | 자소서` 구조로 단순화했습니다. 경험은 원문 메모 우선, 고급 구조화는 저장 후 선택적으로 노출합니다. |
| 문항 경험 선택 단순화 | 완료 | 문항에서는 경험 제목을 최대 3개까지 선택하는 흐름만 노출하고, 스토리·근거 세부 화면은 숨겼습니다. |
| AI 초안 UI 단순화 | 완료 | 문항에서는 AI 초안 생성·초안 확인·답변 적용만 우선 보여주고, 경고는 접어서 확인할 수 있게 했습니다. |
| AI 오류 안내 | 완료 | 경험카드의 AI 분석 흐름과 자소서 AI 초안 흐름 모두 로딩 상태와 실제 오류 문구를 표시합니다. |
| AI 제공자 오류 진단 | 완료 | OpenAI 호출 실패 시 상태 코드와 제공자 메시지를 표시해 API 키·모델·할당량 문제를 구분할 수 있습니다. |
| GPT-5 AI 호출 호환성 | 완료 | GPT-5 계열에서 지원하지 않는 `temperature` 옵션을 제거해 기본 설정으로 요청합니다. |
| 자소서 시작 단순화 | 완료 | 회사명·지원 분야·첫 문항만 필수로 두고, 지원 기간은 선택 입력으로 접었습니다. |

### 다음 개발 우선순위

1. **AI 품질 피드백**: 질문 적합성, 구체성, 팀 성과와 개인 기여 구분, 분량, 중복 표현을 점검합니다.
2. **성과 지표 보완**: 이전값·이후값·단일값·근거·검증 상태와 변화율 계산을 확장합니다.
3. **경험 검색 확장**: 서버 측 검색, 전체 필터, cursor 기반 더 보기, 경험 비교를 구현합니다.

## 테스트 방법

### 1. 정적 검증

```bash
npm install
npm run typecheck
npm run build
```

### 2. 화면만 빠르게 확인

```bash
npm run dev
```

Vite 주소(일반적으로 `http://localhost:5173`)에서 화면과 비로그인 상태를 확인할 수 있습니다. 이 방식은 Cloudflare Functions를 실행하지 않으므로 로그인, 저장 API는 테스트할 수 없습니다.

### 3. 로그인·DB를 포함한 통합 테스트

1. Neon의 **테스트용 DB**에 `schema.sql` 전체를 실행합니다.
2. `.dev.vars.example`을 `.dev.vars`로 복사하고 실제 테스트 환경변수를 입력합니다.
3. Google OAuth 콘솔에 다음을 등록합니다.
   - Authorized JavaScript origin: `http://127.0.0.1:8788`
   - Authorized redirect URI: `http://127.0.0.1:8788/api/auth/callback/google`
4. 실행합니다.

```bash
npm run dev:pages
```

5. `http://127.0.0.1:8788`에서 Google 로그인 후 아래 체크리스트를 수행합니다.

### 수동 체크리스트

- [ ] 로그인 후 내 이름과 이메일이 좌측 하단에 표시된다.
- [ ] 경험 아카이브에서 빠른 메모를 저장하고, 새 카드가 목록에 나타난다.
- [ ] 카드에 상황·과제·행동·결과·성과 수치·역량·스토리 각도를 입력하고 수정할 수 있다.
- [ ] 카드 작성 중 `임시 저장` 후 아카이브로 나갔다가 다시 열면 저장본을 불러올 수 있다.
- [ ] 저장하지 않은 내용을 수정한 뒤 브라우저 탭을 닫거나 새로고침하면 이탈 경고가 표시된다.
- [ ] 보관한 카드는 상태 필터로 확인할 수 있다.
- [ ] 새 자소서를 저장한 뒤 문항의 `경험 찾기`에서 핵심 경험 1개와 보조 경험을 연결할 수 있다.
- [ ] 새 자소서의 `경험카드 활용 문항 예시` 버튼을 누르면 해당 문항이 추가된다.
- [ ] 연결 경험의 스토리 각도를 선택하면 문항에 제목과 핵심 메시지가 표시된다.
- [ ] 경험을 연결하거나 해제해도 작성한 자소서 답변이 바뀌지 않는다.
- [ ] AI 환경변수 설정 후 `AI 초안 만들기`를 누르면 답변 흐름(개요) 입력 모달이 뜨고, 부족하면 추가 질문 후 초안이 생성된다.
- [ ] `답변에 적용` 전에는 기존 답변이 바뀌지 않고, 적용 후 자소서 저장을 눌러야 답변이 갱신된다.
- [ ] 문항에 기업 정보를 입력하고 AI 초안을 생성하면 해당 문항에 기업 정보가 저장된다.
- [ ] 경험카드 저장 후 `AI 분석`을 누르면 모달에서 질문이 하나씩 표시되고, 답변·건너뛰기·그만하기가 모두 동작한다.
- [ ] `AI 분석`에서 답변을 반영하면 경험 원문 메모가 보강되어 갱신된다.

## 주의 사항

- `schema.sql`은 기존 `experiences` 데이터를 삭제하지 않습니다.
- API는 로그인 사용자 소유의 카드와 문항만 읽거나 수정합니다.
- AI는 원문 메모·연결된 사실·사용자가 직접 답변한 내용만 근거로 사용하며, 사실이나 수치를 새로 지어내지 않습니다.
