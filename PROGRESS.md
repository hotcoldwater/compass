## 현재 구현 범위

### 완료 또는 사용 가능

| Phase | 상태 | 구현 내용 |
|---|---|---|
| 0. 기존 기능 안정화 | 부분 완료 | 기존 Google 로그인·자소서 저장을 유지했고 `npm run typecheck`, `npm run build`를 통과합니다. |
| 1. 경험 아카이브 골격 | 완료 | 경험 아카이브, 검색, 카테고리·상태 필터, 카드 목록, 빠른 메모와 단계별 기록 진입을 제공합니다. |
| 2. 경험카드 DB/이전 | 완료 | 카드·성과·역량·스토리 테이블과 사용자 소유권 검증을 추가했습니다. 기존 `experiences`는 보존하고 신규 카드의 원문 메모로 복사합니다. |
| 3. 경험 작성 | 부분 완료 | 기본 정보, 원문 메모, 상황·과제·행동·결과·배운 점, 성과, 역량, 태그를 저장합니다. 브라우저 임시 저장과 저장되지 않은 변경 이탈 경고를 제공합니다. |
| 4. 여러 스토리 각도 | 부분 완료 | 카드별 스토리 각도 생성·저장 및 대표 스토리 표시 기반을 제공합니다. |
| 7. 문항-경험 수동 연결 | 완료 | 문항별 핵심 경험 1개·보조 경험 2개를 연결하고, 스토리 각도와 핵심 사실·확인 수치를 확인할 수 있습니다. |
| 문항 예시 | 완료 | 지원 동기·직무 역량·문제 해결·협업·실패와 개선 예시 문항을 새 자소서 화면에서 바로 추가할 수 있습니다. |

## 테스트 방법

### 1. 정적 검증

```bash
npm install
npm run typecheck
npm run build
```

### 2. 화면만 빠르게 확인

```bash
npm run dev
```

Vite 주소(일반적으로 `http://localhost:5173`)에서 화면과 비로그인 상태를 확인할 수 있습니다. 이 방식은 Cloudflare Functions를 실행하지 않으므로 로그인, 저장 API는 테스트할 수 없습니다.

### 3. 로그인·DB를 포함한 통합 테스트

1. Neon의 **테스트용 DB**에 `schema.sql` 전체를 실행합니다.
2. `.dev.vars.example`을 `.dev.vars`로 복사하고 실제 테스트 환경변수를 입력합니다.
3. Google OAuth 콘솔에 다음을 등록합니다.
   - Authorized JavaScript origin: `http://127.0.0.1:8788`
   - Authorized redirect URI: `http://127.0.0.1:8788/api/auth/callback/google`
4. 실행합니다.

```bash
npm run dev:pages
```

5. `http://127.0.0.1:8788`에서 Google 로그인 후 아래 체크리스트를 수행합니다.

### 수동 체크리스트

- [ ] 로그인 후 내 이름과 이메일이 좌측 하단에 표시된다.
- [ ] 경험 아카이브에서 빠른 메모를 저장하고, 새 카드가 목록에 나타난다.
- [ ] 카드에 상황·과제·행동·결과·성과 수치·역량·스토리 각도를 입력하고 수정할 수 있다.
- [ ] 카드 작성 중 `임시 저장` 후 아카이브로 나갔다가 다시 열면 저장본을 불러올 수 있다.
- [ ] 저장하지 않은 내용을 수정한 뒤 브라우저 탭을 닫거나 새로고침하면 이탈 경고가 표시된다.
- [ ] 보관한 카드는 상태 필터로 확인할 수 있다.
- [ ] 새 자소서를 저장한 뒤 문항의 `경험 찾기`에서 핵심 경험 1개와 보조 경험을 연결할 수 있다.
- [ ] 새 자소서의 `경험카드 활용 문항 예시` 버튼을 누르면 해당 문항이 추가된다.
- [ ] 연결 경험의 스토리 각도를 선택하면 문항에 제목과 핵심 메시지가 표시된다.
- [ ] 경험을 연결하거나 해제해도 작성한 자소서 답변이 바뀌지 않는다.

## 주의 사항

- `schema.sql`은 기존 `experiences` 데이터를 삭제하지 않습니다.
- API는 로그인 사용자 소유의 카드와 문항만 읽거나 수정합니다.
- 현재 AI 기능은 구현하지 않았으므로 경험 사실이나 수치를 자동 생성하지 않습니다.
